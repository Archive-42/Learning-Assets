Tree Traversal

Tree traversal refers to the process of visiting each node in a tree exactly once. The two most common methods of tree traversal are:

  • Depth-First Search
  • Breadth-First Search

Due to the non-linear structure of a tree (from a given node, there is more than one possible next node), some nodes must be deferred—stored in some way for later visiting. This is often done via a stack (LIFO) or queue (FIFO).

As a tree is a self-referential data structure, traversal can be done by recursion or corecursion; in these cases the deferred nodes are stored implicitly in the call stack.

Depth-first search is easily implemented via a stack, including recursively (via the call stack), while breadth-first search is easily implemented via a queue, including corecursively.
